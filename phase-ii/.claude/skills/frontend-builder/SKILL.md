Skill name: Frontend Skill

Purpose:
Enable the design and implementation of frontend user interfaces, including
pages, components, layouts, and styling, in a consistent and maintainable way.

Scope:
This skill governs how frontend UI is structured, rendered, and styled.
It focuses on clarity, responsiveness, and component-based architecture.

Capabilities:
- Build application pages and routes
- Create reusable UI components
- Design layouts using modern frontend patterns
- Apply responsive and accessible styling
- Manage component composition and hierarchy
- Separate server and client components when applicable
- Implement basic frontend state and interaction patterns

Design principles enforced:
- Component-driven architecture
- Clear separation of concerns
- Responsive-first design
- Accessibility by default
- Consistent styling conventions
- Minimal and readable UI logic

Input handling:
- Accepts UI requirements and layout definitions
- Translates specifications into pages and components
- Validates component boundaries and responsibilities

Constraints:
- Frontend layer only
- No backend or database logic
- No authentication internals
- No speculative UI features or redesigns

Dependencies:
- Design specifications or UI requirements
- Backend API contracts (when data is required)

Output standards:
- Clean, modular component structure
- Responsive layouts across devices
- Semantic and accessible markup
- Consistent styling
- Maintainable frontend code

Success definition:
- Pages and components render correctly
- Layouts are responsive and accessible
- Styling is consistent and clear
- UI behavior matches specifications
- Frontend code is readable, modular, and extensible
